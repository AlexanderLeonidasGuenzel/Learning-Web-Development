Units & Sizes

Absolute Value

Pixels are an absolute value. It is better not to use a fixed font-size.

html {
   font-size: 16px;
}

Browsers have a default font-size of 16px. In the browsers settings the font-size can be adjusted. If the website has an absolute value for the font-size, browser adjustments can't change the font-size preferences.

Relative Values

With using relative units wisely, you can make elements to scale relatively to everything else on the page.

Unit Relative to
em Font size of the parent...
rem Font size of the root element.
vw 1% of the viewport's width.
vh 1% of the viewport's height

Percentages

The h1 element has a default width of 100%. If you change it to 50% like this, it will take up half of the width of the body.

h1 {
   width: 50%
}
<body>
  <h1> Heading </h1>
</body>

If we put the h1 element into an header with an own width of 50%, it will affect the width of the h1 element. In total it will be just 25% of the body width.

header {
   width: 50%
}
h1 {
   width: 50%
}
<body>
   <header>
     <h1> Heading </h1>
  </header>
</body>

Remember: Setting a percentage for the width means that you are making it relative to the parent. It's only using what it has available to it.

rem

This is the unit that is typically used for font size and it relates to the font size of the root (html) element

If the browser has the default value of 16px, than the paragraph has an font-size of 32px.

p {
   font-size: 2rem;
}

This has no effect because the font size of the paragraph looks for the root element, not the parent (main).

main {
   font-size: 2rem;
}

p {
   font-size: 2rem;
}

em

Now the font size of the paragraph becomes 64px, because the paragraph doubles the font size of its parent the main element.

main {
   font-size: 2rem;
}

p {
   font-size: 2em;
}

But em is better for margin or padding. Em for font size can suddenly amplify to much.

Viewport Height - vh

When we want the body element to be the full height of the page even if we don't have enough content. Using the height: 100vh, would limit the page and the content could outgrow. Taking min-height property allows teh viewport to grow with the content and at the same time it is the full page.

body {
  min-height: 100vh;
}
